OLÁ MUNDO DA PROGRAMAÇÃO
Por que Programar?
Antes de mais nada,
Bem-vindo ao mundo onde você ensina o computador a pensar.
  • Resolver problemas do mundo real.
  • Criar jogos, apps e robôs
  • Automatizar tarefas
  • Dar "inteligência" às máquinas
O que é Programar?
  • Dar instruções para um computador
  • Usar uma linguagem que a máquina entenda
  • Resolver problemas passo a passo
Elementos da Programação
  • Linguagem: forma de comunicar
  • Código-fonte: instruções escritas
  • Algoritmo: passo a passo lógico
A programação é baseada no uso de Algoritmos
 O que é um ALGORITMO?
Os algoritmos são uma sequência de instruções detalhadas passo a passo.
  • São escritos por programadores para que uma máquina execute uma tarefa, como se fosse uma receita.
  • A máquina deve seguir todas as tarefas exatamente como escrito.
  • Um software é um conjunto de diferentes algoritmos, cada um com seu propósito
  • Em todo sistema robótico, seguimos esse ciclo:

Entrada → Processamento → Saída
Estrutura Básica dos Algoritmos

Todo algoritmo segue uma lógica de pensamento organizada em três partes principais:


🟦 Sequência
As ações acontecem uma depois da outra.
Ex: acender a luz → ligar o ventilador → sentar.

🟧 Repetição
Um bloco de ações que se repete enquanto uma condição for verdadeira.
Ex: repetir 10 vezes → piscar o LED.

🟨 Condição (seleção)
O algoritmo decide o que fazer dependendo de uma situação.
Ex: se o botão for pressionado → mover o robô.
Sequência na Programação
O algoritmo sempre executa de forma sequencial em uma única direção.
Ele percorre cada passo sem retornar ao anterior — como um rio que flui em linha reta.

Exemplo de Sequência
  • 1️⃣ Ligar o computador
  • 2️⃣ Abrir o navegador
  • 3️⃣ Acessar o site do Blockly
  • 4️⃣ Resolver o desafio
Repetição na Programação
Quando identificamos um PADRÃO, podemos repetir comandos! 😄
Chamamos isso de um loop ou laço de repetição.

Exemplo de Repetição
Repetir 5 vezes:
  • 1️⃣ Acender LED
  • 2️⃣ Esperar 1 segundo
  • 3️⃣ Apagar LED
  • 4️⃣ Esperar 1 segundo
Condições: Tomando Decisões
Condições são situações em que o caminho de execução tem mais de uma opção
Isso permite que a máquina avaliar se deve executar ou não.
Tornando nosso código mais autônomo.

Exemplo de Condição
Se botão pressionado:
  • Mover servo para 120°
Senão:
  • Mover servo para 90°
Variáveis
  • Espaço para guardar dados
  • Podem mudar de valor
  • Ex: idade = 12
  • Possuem vários tipos
  • Servem para controlar valores durante a execução
Tipos de Dados
Existem muitos tipos, os mais comuns são:
  • Número (10, 3.5)
  • Texto ("Olá mundo")
  • Lógico (true/false)
  • ... e mais ...
Verdadeiro ou Falso
  • O tipo de dado lógico só pode ter valores: Verdadeiro (True) ou Falso (False). (não existe outra opção)
  • É usado para Tomada de decisões em programas
  • Ao combinar condições diferentes, criamos decisões complexas usando Operadores Lógicos.
Se (a senha digitada é correta) { -> ENTRAR /* condição é verdadeira */ } senão { -> ACESSO NEGADO /* condição é falsa */ }
Introdução a Lógica Booleana
  • Na matemática, conhecemos muitos operadores aritméticos: soma, subtração, multiplicação e divisão.
  • Também já vimos os operadores de relação: igual, diferente, maior, menor.
  • Estes operadores funcionam com números. Já os operadores lógicos são usados para combinar condições, formando resultados verdadeiros (V) ou falsos (F).
Conectivo E (AND)
  • Imagine a situação: seu pai diz:
    “no Natal, eu te darei um vídeo game e uma bicicleta”.
  • Se ele te der apenas um dos dois, a promessa será falsa.
  • Uma condição com o conectivo "E" só será verdadeira quando ambas as condições são cumpridas.
Conectivo OU (OR)
  • Mas desta vez, seu pai disse:
    “no Natal, eu te darei um vídeo game ou uma bicicleta”.
  • Se ele te der apenas um dos dois, a promessa será verdadeira.
  • O “OU” é falso apenas quando nenhuma das partes acontece.
Operador NÃO (NOT)
  • O operador Não inverte o valor lógico de uma condição:
  • Se algo é verdadeiro, NÃO torna falso — e vice-versa.
  • A: Verdadeiro;
  • ¬A: Falso;
Erros de sintaxe e erros de semântica
  • A máquina entende apenas zeros e uns, para programarmos uma máquina, precisamos fazer uso de uma Linguagem de Programação
  • Toda linguagem de programação tem uma gramática assim como nossa língua
  • Erro de Sintaxe ocorrem por uso incorreto de estruturas ou erros de digitação
  • Nos erros de Semântica, o código executa, mas produz resultados incorretos porque a lógica está errada.
